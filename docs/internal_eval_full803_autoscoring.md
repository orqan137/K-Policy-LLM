# 자체 평가 전체 803문항 자동채점 결과

작성일: 2026-06-16

## 요약

- 이번 결과는 기존 `limit_per_task=20` 제한 평가가 아니라, 현재 target_eval 후보 문항 803개 전체를 네 단계에 대해 다시 생성·채점한 결과다.
- 채점 기준은 `target_eval_strict_v2`이다. 요구 구획이 없는 문항에는 구획 점수를 자동으로 부여하지 않고, 단답 문항은 짧다는 이유만으로 감점하지 않도록 길이 조건을 분리했다.
- 이 평가는 공개 객관식 벤치마크나 사람 정성평가가 아니라, 프로젝트 내부 목표 작업에 대한 규칙 기반 자동채점 진단이다.

## 기존 평가에서 바로잡은 점

기존 제한 평가는 보고서 초안 작성 중 빠르게 단계별 경향을 확인하기 위해 문항 유형별 최대 20개만 뽑아 실행한 진단이었다. 이때 사용한 원래 자동채점식은 전체 추세를 보는 데는 쓸 수 있었지만, 보고서에 그대로 공개하기에는 세 가지 문제가 있었다.

첫째, 요구 구획이 없는 문항도 `섹션 충족률`을 1점으로 처리했다. 이 때문에 단답, 개념 설명, 현재 사실 확인처럼 애초에 답변 구획을 요구하지 않은 문항이 구획 점수 35점을 자동으로 받았다. 이는 실제 답변 품질보다 점수를 높게 보이게 할 수 있었다.

둘째, 모든 문항에 같은 길이 조건을 적용해 답변 길이가 20자 미만이면 감점했다. 그러나 "서울입니다."처럼 의도적으로 짧게 답해야 하는 단답 문항은 짧은 답변이 오히려 정답 형식이다. 따라서 단답형 문항에는 별도 길이 조건을 적용하는 것이 맞다.

셋째, 전체 후보 문항은 803개인데 보고서 초안에는 `limit_per_task=20` 제한 평가로 생성한 139개 결과와, 그중 6개 항목으로 묶은 123개 결과가 함께 설명되어 문항 수가 혼동될 수 있었다.

이번 재평가에서는 이 부분들을 수정했다. 요구 구획이 없는 문항은 구획 점수를 계산에서 제외하고, 남은 구성 요소의 가중치를 다시 정규화했다. 또한 단답형 또는 기준답변이 짧은 문항은 2자 이상 800자 이하이면 길이 조건을 통과하도록 바꾸었다. 마지막으로 803개 전체 문항을 모두 생성·채점하고, 기존 제한 평가와 구분되도록 `full803_strict` 산출물로 분리했다. 따라서 이번 결과는 기존 제한 평가 점수와 직접 같은 숫자로 비교하기보다, 더 엄밀하게 다시 산출한 내부 자동채점 결과로 보는 것이 맞다.

## 자동채점 수식

점수는 0~1 범위로 계산한 뒤 100을 곱해 표시했다. 사람이 읽고 점수를 매긴 것이 아니라, 각 문항에 정의된 요구 구획, 필수어, 금지어, 수치 정답, 단위 조건을 자동으로 대조했다.

텍스트형 문항의 기본 구성 요소는 다음과 같다.

`텍스트형 점수 = 정규화(0.35 × 요구 구획 충족률 + 0.25 × 필수어 충족률 + 0.25 × 금지어 회피 + 0.15 × 길이 조건)`

수치형 문항의 기본 구성 요소는 다음과 같다.

`수치형 점수 = 정규화(0.45 × 수치 정답 + 0.15 × 단위 포함 + 0.15 × 필수어 충족률 + 0.15 × 금지어 회피 + 0.10 × 길이 조건)`

여기서 `정규화`는 해당 문항에 실제로 적용되는 구성 요소만 사용한다는 뜻이다. 예를 들어 요구 구획이 없는 단답 문항은 요구 구획 항목을 제외하고, 필수어·금지어·길이 조건만으로 점수를 계산한다.

| 구성 요소 | 적용 문항 | 의미 |
| --- | --- | --- |
| 요구 구획 충족률 | 텍스트형 중 요구 구획이 있는 문항 | 안건/문맥, 쟁점, 근거, 한계처럼 문항이 요구한 답변 구획이 포함된 비율 |
| 필수어 충족률 | 텍스트형·수치형 | 문항별 핵심 용어 또는 필수 표현이 포함된 비율 |
| 금지어 회피 | 텍스트형·수치형 | 금지 표현이나 자료 밖 단정이 없으면 1점, 있으면 0점 |
| 길이 조건 | 텍스트형·수치형 | 일반 문항은 20~2,500자, 단답형 문항은 2~800자이면 1점 |
| 수치 정답 | 수치형 | 출력의 숫자가 정답값과 허용 오차 안에 있으면 1점 |
| 단위 포함 | 수치형 | `%p`, `억 원` 등 문항이 요구한 단위가 포함되면 1점 |

## 해석

전체 평균은 사전학습 베이스라인 35.6점, 추가학습(CPT) 35.5점, 지도학습(SFT) 36.1점, 선호학습(DPO) 36.5점이다. 절대 점수가 낮은 이유는 이번 채점이 문항별 필수어, 금지어, 수치 정답, 단위, 답변 구획을 엄격히 대조하기 때문이다. 특히 수치형 문항 301개가 전체의 큰 비중을 차지하고, 네 단계 모두 표 수치 계산에서 낮은 점수를 보였다.

단계별 변화는 제한 평가와 같은 방향이다. CPT는 추가 사전학습에 가까워 지시 형식이나 수치 계산을 직접 개선하지 못했고, 전체 평균도 사전학습 베이스라인보다 소폭 낮다. SFT는 단답, 정책 설명, 근거 부족 응답처럼 답변 형식이 있는 문항에서 일부 개선을 만들었다. DPO는 전체 평균에서 가장 높지만 개선 폭은 작다. 이는 DPO가 모든 내부 목표 작업을 크게 끌어올렸다기보다, 반복 억제와 실패 응답 교정에 초점을 둔 보강쌍이 일부 항목에서 점수를 보탠 것으로 해석하는 편이 안전하다.

보고서에는 이 결과를 공식 성능 지표로 쓰기보다, 사후학습이 목표 작업에서 어떤 방향의 개선을 만들었는지 보여주는 내부 진단 결과로 제시해야 한다. 공개 지표는 KMMLU, CLIcK, KoBEST 등 별도 표와 구분해 써야 한다.
