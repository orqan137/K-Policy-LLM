# 보고서용 전체 실험 기록

작성일: 2026-06-16

이 문서는 보고서 작성에 필요한 실험 흐름, 수치, 판단 근거, 한계, 표현 기준을 한곳에 모은 정리본이다. 실행 경로, 긴 run id, 로그 파일 위치처럼 보고서 본문에 도움이 되지 않는 운영 정보는 제외하였다.

## 1. 최종 요약

본 프로젝트는 한국 정치·법률·정책 자료를 이해하고 설명하는 도메인 특화 LLM을 처음부터 구축하는 것을 목표로 진행하였다. 공개 사전학습 모델을 가져와 미세조정하는 방식이 아니라, 토크나이저, 모델 구조, 사전학습 데이터팩, 사후학습 데이터팩, 평가 체계를 직접 구성하였다.

최종 보고서에서 주장할 수 있는 핵심은 다음과 같다.

- 모델은 약 1.221B 파라미터 규모의 dense decoder-only causal LM으로 구성하였다.
- 한국어 중심 80K SentencePiece BPE 토크나이저를 사용하였다.
- 최종 사전학습 데이터팩은 31.21B 토큰 규모이며, 실제 공식 기준 베이스는 토큰학습량 15B 지점으로 확정하였다.
- 16B 지점은 실패로 폐기한 것이 아니라, 일부 일반 지표는 소폭 나아졌지만 한국어 확장 지표가 15B보다 낮아 최종 기준 베이스에서 제외하였다.
- 사후학습은 CPT, SFT, DPO 순으로 수행하였다.
- GRPO용 수치 보상 데이터는 제작했지만 최종 실행 계보에는 포함하지 않은 후속 후보이다.
- 공개 벤치마크 절대 점수는 공개 한국어 특화 LLM과 비교해 낮다. 공개 지표는 경쟁 성능 근거가 아니라 토큰학습량 선택과 단계별 변화 확인용 진단 지표로 해석해야 한다.
- 자체 평가는 803개 내부 생성형 진단 문항을 네 단계 모두에서 생성·채점한 규칙 기반 자동채점 결과이다. 사람 점수 평가는 수행하지 않았다.
- 개선 사례는 자동채점 후보 중 질문, 기준 답변, 단계별 출력을 GPT-5.5로 대조해 선별한 사례이며, 별도 사람 정성평가 점수는 아니다.

## 2. 목표 작업 정의

최종 목표는 단순히 정치 문장을 잘 생성하는 것이 아니라, 정치·법률·정책 문서를 읽고 사용자가 이해하기 쉬운 구조로 바꾸는 것이다.

| 목표 작업 | 설명 |
| --- | --- |
| 정치 의제 이해 보조 | 국회·지방의회 회의록에서 주요 발언, 안건, 쟁점, 결정 여부를 정리 |
| 법률안·검토보고서 해설 | 법률안의 목적, 개정 내용, 검토 포인트, 쟁점, 한계를 설명 |
| 정책자료·공약 문서 분석 | 정당정책, 선거공약, 정책보고서의 방향과 차이를 분리 |
| 정치·법률 쟁점 구조화 | 찬성 논거, 반대 또는 우려 논거, 법적 쟁점, 재정·행정 고려사항을 구분 |
| 근거 기반 질의응답 | 제공 문서 안에서 확인 가능한 근거를 바탕으로 답하고, 자료 밖 내용은 구분 |
| 표·수치 자료 해석 | 투표율, 예산, 의석수, 연도별 변화의 값·차이·단위를 설명 |
| 기본 정치·법률 개념 설명 | 국회 절차, 법률안 처리 단계, 선거제도, 헌법·행정 기본 개념을 짧게 설명 |

## 3. 모델 설계 기록

| 항목 | 설정 | 판단 |
| --- | --- | --- |
| 모델 유형 | Llama 계열 dense decoder-only causal LM | 생성과 사후학습 적용이 단순하고 실패 원인 분리가 쉬움 |
| 파라미터 규모 | 약 1.221B | 0.46B보다 표현력이 높고 3B 이상보다 학습 가능성이 높음 |
| 레이어 | 28 layers | 1.2B급에서 깊이와 비용의 균형 |

| FFN 차원 | 4,096 | gated FFN 구성 |
| 어텐션 | 16 heads, KV heads 8, GQA | key/value 저장 부담을 줄임 |
| head dimension | 128 | hidden size와 head 수에 맞춤 |
| 활성화·정규화 | SiLU, RMSNorm | Llama 계열에서 검증된 구성 |

| 문맥 길이 | 4,096 tokens | 회의록·법률안 일부 문맥을 담되 학습 비용을 감당 가능한 범위로 제한 |
| 토크나이저 | SentencePiece BPE, vocab 80,000 | 한국어 복합어와 법률·정책 용어의 과도한 분절을 줄이기 위함 |
| 정밀도 | bf16 | A6000·H200 환경의 속도와 안정성 균형 |
| 임베딩 | tie word embeddings | 파라미터 절감 |

## 6. 사전학습 기록

사전학습은 다음 토큰 예측 방식으로 수행하였다. 입력 데이터는 4,096토큰 길이의 학습 입력으로 사전 변환하여 사용하였다. 체크포인트에는 모델 가중치뿐 아니라 옵티마이저 상태, 학습률 스케줄러, 난수 상태, 데이터 읽기 위치 정보까지 저장하여 장애 발생 시 가능한 한 이어서 재개할 수 있도록 했다.

초기에는 모델 파라미터 수만 키우면 성능이 향상될 것이라고 보았지만, 실제로는 모델 규모와 토큰학습량의 균형이 더 중요했다. Chinchilla 스케일링 법칙 관점에서도 파라미터 수에 비해 토큰학습량이 부족하면 문장 형식은 일부 습득하더라도 지식 유지, 문맥 처리, 수치 처리에서 불안정해질 수 있다.

## 7. 15B 기준 베이스 선택 기록

사전학습팩은 31.21B 토큰 규모였지만, 실제 최종 기준 베이스는 토큰학습량 15B 지점으로 선택하였다. 이 결정은 확보된 토큰 전체를 끝까지 학습시키는 방식이 아니라, 중간 체크포인트별 공개 지표를 비교한 결과에 기반한다.

한국어 확장 지표의 가중 평균은 다음 흐름을 보였다.

| 토큰학습량 | 한국어 확장 가중 평균 |
| ---: | ---: |
| 1.5B | 26.00% |
| 6B | 28.06% |
| 9.5B | 28.06% |
| 13.5B | 28.06% |
| 15B | 28.59% |
| 16B | 27.52% |

15B와 16B의 추가 비교는 다음과 같다.

| 평가 범위 | 15B weighted | 16B weighted | 해석 |
| --- | ---: | ---: | --- |
| MMLU/MMLU-Pro 전체 | 18.06% | 18.22% | 16B가 0.16%p 높음 |
| 영어 확장 평가 | 29.10% | 29.08% | 거의 동일 |
| 한국어 확장 평가 | 28.59% | 27.52% | 15B가 1.07%p 높음 |
| 전체 확장 평가 | 28.72% | 27.90% | 15B가 0.82%p 높음 |

따라서 16B는 폐기된 실패 체크포인트가 아니라 비교 실험 후보로 남겼다. 다만 한국어 목표 지표가 15B보다 낮아 최종 보고서의 사전학습 기준 베이스는 15B로 정했다. 24B까지 학습하지 않은 이유도 같은 흐름으로 설명해야 한다. 단순히 자원이 부족해서 포기한 것이 아니라, 중간 평가에서 한계효과 둔화와 한국어 지표 하락을 확인해 사후학습과 실패사례 보강 쪽의 기대효용이 더 크다고 판단한 것이다.

### 8.3 DPO 보강 기록

15B 사전학습 베이스에서 출발한 SFT 모델을 기준으로 실패 응답을 수집해 DPO 보강을 시도하였다.

주요 기록은 다음과 같다.

- 실패사례 기반 DPO source prompt 10,000개를 생성하였다.
- 모델이 낸 completion 10,000개를 확보하였다.
- bad JSON과 empty completion은 0건이었다.
- DPO pack은 9,000개 학습, 1,000개 검증으로 구성하였다.
- 구성은 SFT split model failures 6,388개, 기존 준비 DPO row 3,536개, 직접 제작 anchor 76개였다.
- quick prompt와 untouched natural prompt는 DPO pack과 exact overlap이 0개가 되도록 분리하였다.
- 온도와 메모리 문제로 큰 batch/긴 sequence 설정은 중단되었고, 최종적으로 seq_len 512, micro batch 8의 100-step chunk가 안전하게 완료되었다.
- 이후 추가 이어달리기 형태의 DPO checkpoint를 만들었고, 보고서에는 DPO가 모든 항목을 일괄 개선했다고 쓰지 않는 방향으로 정리하였다.

## 9. 공개 벤치마크 기록

### 9.1 평가 방식

공개 벤치마크는 zero-shot next-token 선택지 문자 정확도로 평가하였다. 즉 A/B/C/D 같은 선택지 문자에 대한 다음 토큰 로그확률을 비교해 가장 높은 선택지를 예측값으로 삼았다.

중요한 점은 이 방식이 EXAONE, A.X 등 공개 모델 기술보고서의 평가 방식과 다를 수 있다는 것이다. 예를 들어 EXAONE 3.0은 KMMLU를 5-shot accuracy로, KoBEST 계열을 5-shot F1로 보고한다. 본 프로젝트의 점수는 동일 조건 순위표가 아니라, 우리 모델의 단계별 변화 확인용 진단 지표이다.

### 9.3 공개 지표 해석

공개 지표의 절대 점수는 낮다. 특히 4지선다형에서 25% 근처, 2지선다형에서 50% 근처는 무작위 기준선에 가깝다. 따라서 KoBEST 일부 항목이 50% 안팎이라는 사실을 성능 확보로 해석하면 안 된다.

보고서에서 안전한 해석은 다음과 같다.

> 공개 한국어 벤치마크에서 절대 정답률은 공개 한국어 특화 LLM과 비교해 낮은 수준에 머물렀다. 특히 KoBEST 일부 항목은 무작위 선택 기준선에 가까워, 본 실험의 공개 지표 결과를 경쟁 성능으로 해석하기는 어렵다. 따라서 본 보고서에서는 공개 지표를 모델 완성도의 근거라기보다 사전학습 토큰학습량 선택과 사후학습 단계별 변화 확인을 위한 진단 지표로 사용하였다.

피해야 할 표현은 다음과 같다.

- 공개 벤치마크에서 경쟁력을 확인하였다.
- 공개 한국어 모델과 유사한 수준에 도달하였다.
- KoBEST 50점대로 성능이 확보되었다.
- DPO가 공개 지표 전반을 개선하였다.

### 10.3 자동채점 기준

텍스트형 문항의 기본 점수는 다음 요소를 사용하였다.

`텍스트형 점수 = 정규화(0.35 × 요구 구획 충족률 + 0.25 × 필수어 충족률 + 0.25 × 금지어 회피 + 0.15 × 길이 조건)`

수치형 문항의 기본 점수는 다음 요소를 사용하였다.

`수치형 점수 = 정규화(0.45 × 수치 정답 + 0.15 × 단위 포함 + 0.15 × 필수어 충족률 + 0.15 × 금지어 회피 + 0.10 × 길이 조건)`

여기서 정규화는 해당 문항에 실제로 적용되는 구성 요소만 사용한다는 뜻이다. 요구 구획이 없는 문항에는 구획 점수를 자동으로 부여하지 않았다. 단답형 또는 기준답변이 짧은 문항은 20자 미만이라는 이유만으로 감점하지 않고, 2자 이상 800자 이하를 길이 조건 통과로 보았다.

## 11. 사후학습 개선 사례 기록

보고서에 사용한 여섯 사례는 모두 사전학습 베이스라인과 CPT 단계에서 실패한 질의이다. 사례 1~4는 SFT에서 처음 개선 판정을 받았고 DPO에서도 유지되었다. 사례 5~6은 SFT에서도 실패했으나 DPO에서 개선 판정을 받았다.

| 사례 | 질문 유형 | 개선 단계 | 핵심 의미 |
| --- | --- | --- | --- |
| 1 | 대한민국 수도 단답 | SFT | 장황한 기사체를 짧은 직접 답변으로 전환 |
| 2 | 기준일이 있는 미국 대통령 질문 | SFT | 질문과 무관한 문장을 멈추고 기준일 포함 단답으로 전환 |
| 3 | 잘못된 반복 답변과 예산 차이 계산 교정 | SFT | 반복 문제를 지적하고 계산식·단위 포함 정답 제시 |
| 4 | 정의당 중립 소개 | SFT | 정당 설명에서 중립성·최신성 한계 반영 |
| 5 | 세제·재정정책 설명 | DPO | SFT의 근거 없는 최신성 단정을 줄이고 비교 기준 제시 |
| 6 | 보건의료정책과 법률안·예산·회의록 연결 | DPO | 자료 유형별 역할과 확인 기준을 맞춰 제시 |

개선 사례의 선별 기준은 다음과 같다.

- 사전학습 또는 CPT 출력이 질문과 무관하거나 반복·환각·기사체로 실패했는가
- SFT 또는 DPO 출력이 질문에 직접 답했는가
- 기준 답변과 비교했을 때 핵심 요구를 충족했는가
- SFT가 해결한 유형과 DPO가 추가로 해결한 유형이 구분되는가
- 보고서 목표인 반복 억제, 근거 없는 단정 감소, 단답 직접성, 정책·법률 문서 구조화 개선을 보여주는가
- 자동채점 점수 흐름과 실제 출력 검토가 크게 어긋나지 않는가

이 선별은 사람 평가 점수가 아니라 GPT-5.5를 이용한 사례별 원문 대조 검토이다.

## 12. 실패와 한계 기록

### 12.1 사전학습 한계

- 31.21B 토큰 데이터팩을 확보했지만 최종 기준 베이스는 15B로 멈추었다.
- 16B와 24B 목표는 실패로 폐기한 것이 아니라, 중간 평가상 한계효과 둔화와 한국어 지표 하락 때문에 기준 계보에서 제외하였다.
- 사전학습팩에서 공식 정치·법률 원천 비율은 약 0.586%로 낮았다.
- FineWeb2 policy-core가 11.86%를 차지하지만, 이는 정책 키워드가 포함된 웹 문서이지 공식 정책자료 자체는 아니다.

### 12.2 사후학습 한계

- CPT는 도메인 문체와 용어 보강에 가깝고, 지시응답·정답성 개선을 직접 보장하지 않았다.
- SFT는 답변 형식 안정화와 단답 직접성에는 도움이 되었지만, 공개 선택지형 지표를 일괄적으로 끌어올리지는 못했다.
- DPO는 반복 억제와 근거 없는 단정 제거에 도움이 되는 사례가 있었지만, 정책·공약 비교와 표 수치 해석에서는 SFT보다 낮은 점수가 나왔다.
- 숫자 계산은 DPO만으로 해결하기 어렵고, 별도 산술 SFT, verifier, 보상학습 또는 도구 사용 전략이 필요하다.

### 12.3 평가 한계

- 공개 벤치마크는 zero-shot next-token 선택지 문자 accuracy로 평가했으므로, 공개 모델 보고서의 5-shot accuracy/F1과 직접 비교하면 안 된다.
- 자체 평가는 내부 생성형 자동채점이다. 문항별 필수어, 금지어, 수치 정답, 단위 조건을 엄격히 대조하기 때문에 의미상 괜찮은 답변도 낮게 채점될 수 있다.
- 자체 평가에는 사람 점수 평가가 포함되지 않았다.
- 개선 사례는 자동채점 후보를 GPT-5.5가 원문 대조 검토한 사례이지, 사람 정성평가 표본은 아니다.

## 13. 최종 문서 점검 기록

최종 v11 문서는 다음 사항을 확인하였다.

- 공개 한국어 지표 11개 수치가 기준 CSV와 일치한다.
- 자체 평가 803문항 수치가 최신 full803 strict 결과와 일치한다.

- DPO는 전체 개선이 아니라 일부 항목 개선과 일부 항목 하락을 함께 서술하였다.
- 남은 문제는 일부 띄어쓰기와 표현 다듬기 수준이다.

- 자체 평가는 사람이 평가한 정성평가이다.
- 803개 후보 문항만 만들었고 실제 평가는 일부만 했다.
- GRPO를 최종 성능 개선 단계로 사용했다.
- 16B는 실패해서 폐기했다.
- 24B를 하지 못한 것은 단순 시간 부족 때문이다.

## 16. 최종 판단

보고서의 가장 안전한 방향은 "성능 좋은 최종 모델"을 주장하는 것이 아니라, "도메인 특화 LLM 구축 과정의 실험 기록과 실패 분석, 그리고 15B 기준 베이스 선택과 사후학습 보강의 근거"를 제시하는 것이다.

현재 기록으로 주장 가능한 성과는 다음이다.

- from-scratch 학습 계보를 끝까지 구성하였다.
- 80K 한국어 중심 토크나이저와 31.21B 토큰 사전학습팩을 구축하였다.
- 토큰학습량별 공개 지표 비교를 통해 15B 기준 베이스를 선택하였다.
- CPT, SFT, DPO 사후학습 계보를 구성하였다.
- 공개 지표와 내부 생성형 평가를 분리해 해석하였다.
- 자동채점 803문항 전체 평가를 네 단계 모두에 대해 수행하였다.
- 사전학습 실패 질의가 SFT 또는 DPO에서 개선되는 사례를 확인하였다.
- 남은 약점과 후속 개선 방향을 숨기지 않고 제시하였다.
